# タスク完了時のチェックリスト

コードの変更を行った後、以下の手順を実行してください：

## 1. コード品質チェック

### リンティング
```bash
npm run lint
```
- ESLintのエラーや警告がないことを確認

## 2. テスト

### テストの実行
```bash
npm test
```
- すべてのテストがパスすることを確認
- 新機能を追加した場合は、対応するテストを追加

### テストカバレッジ
```bash
npm run test:coverage
```
- 必要に応じてカバレッジを確認

## 3. ビルドチェック

```bash
npm run build
```
- ビルドエラーがないことを確認
- TypeScriptの型エラーがないことを確認

## 4. 動作確認

```bash
npm run dev
```
- 開発サーバーで実際の動作を確認
- 追加・変更した機能が正しく動作することを確認
- UIの見た目に問題がないことを確認
- ダークモードでも正しく表示されることを確認（該当する場合）

## 5. コミット前の最終チェック

- [ ] リンティングエラーがない
- [ ] すべてのテストがパス
- [ ] ビルドが成功
- [ ] 不要なコメントやconsole.logが残っていない
- [ ] 環境変数や機密情報がコミットされていない
- [ ] 適切なコミットメッセージを作成

## 注意事項

- `.env.local` は絶対にコミットしない
- `node_modules/`, `.next/` はgitignoreに含まれている
- 認証情報やAPIキーは環境変数で管理
